# Proce 개발 계획서 (Frontend 중심)

본 계획서는 기획·문서 우선 전략에 따라 UI 스켈레톤과 핵심 흐름을 단계적으로 구축합니다.

---

## 1. 프로젝트 개요
- **프로젝트명**: Proce
- **목표**:
  - 문서 100% → UI 스켈레톤 → 핵심 흐름 검증
  - NoMeet/Policy/Input/Dashboard 4축 정보 구조 확립
  - Promptor DS 일관 적용 및 접근성 기준 준수
- **개발 기간(초안)**:
  - Phase 1 (MVP-UI): 2~3주
  - Phase 2 (IA 고도화): 2~3주
  - Phase 3 (최적화/운영): 2주

---

## 2. 범위
- 포함: 프론트엔드 UI/플로우, 문서화, 디자인 시스템 기본 토큰
- 제외(현 단계): 백엔드, 인증/보안 헤더, 배포 자동화(후속)

---

## 3. 성공 기준
- 문서-코드 일치율 ≥ 95%
- ESLint/Prettier 경고 0
- 핵심 페이지 4종 스켈레톤 구현 및 내비게이션 동작

---

## 4. 일정 (Phase)

### Phase 1 — UI 스켈레톤 (주차 1~2)
- 문서 정제: 서비스 기획/페이지 정의/디자인 시스템 동기화
- 라우팅/프로바이더/레이아웃 셸 구축
- 대시보드, NoMeet, Policy, Input 페이지 골격 배치

### Phase 2 — IA 고도화 (주차 3~4)
- NoMeet 상세 컴포넌트(Briefing/Feedback/Proposal) 인터랙션 목업
- Policy DSL 에디터 목업, 테스트 러너 프레임
- 자유 입력 → 정규화 미리보기 UX 목업

### Phase 3 — 최적화/운영 (주차 5~6)
- 접근성/A11y 점검, 성능 최적화(코드 스플릿)
- 운영 대시보드 위젯/지표 카드 정리
- 문서-코드 최종 동기화 및 릴리즈 노트

---

## 5. 리소스
- Frontend: 1~2
- PM/기획: 1
- 디자인: 1 (파트타임)

---

## 6. 리스크 관리
- 변경 폭 확대 → 문서 우선 합의, 주간 리뷰로 범위 통제
- AI 결과 신뢰성 → 근거/신뢰도 노출, 휴먼 인 더 루프 유지

---

## 7. KPI
- 회의/보고 시간 절감 가시화(목업 단계선 서술형)
- 의사결정 리드타임 단축 경험 설계(플로우 기준)

---

## 8. 부록
- 연동(슬랙/노션/지라) 및 백엔드/보안은 별도 계획서로 분리
