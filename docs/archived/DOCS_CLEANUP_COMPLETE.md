## 🔄 앞으로의 문서 관리

### 새 문서 생성 시
```
1. 정말 필요한 문서인가? 검토
2. 기존 문서에 추가할 수 없나? 확인
3. Root에 둘 만큼 중요한가? 판단
4. 아니면 docs/ 또는 proce_frontend/docs/에 배치
```

### 문서 업데이트 시
```
1. 최신 정보로 유지
2. Outdated 정보는 삭제
3. 중복 내용은 통합
4. 명확하고 간결하게 작성
```

### 문서 삭제 시
```
1. 정말 필요 없는 문서인가? 재확인
2. 혹시 나중에 필요할 수 있다면 Archive
3. 완전히 불필요하면 삭제
```
