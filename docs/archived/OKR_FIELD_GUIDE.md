# 📊 OKR 페이지 - 입력 필드 가이드

> **페이지:** `http://localhost:5173/app/okr`  
> **기능:** Add New Objective (새로운 목표 추가)

---

## 🎯 OKR이란?

**OKR (Objectives and Key Results)** 은 목표 설정 프레임워크입니다.

- **Objective (목표):** 달성하고자 하는 정성적인 목표
- **Key Results (핵심 결과):** 목표 달성을 측정하는 정량적 지표

**예시:**
```
Objective: "고객 만족도를 획기적으로 개선한다"
  ├─ KR1: Net Promoter Score(NPS)를 50점 달성
  ├─ KR2: 고객 이탈률을 5% 이하로 감소
  └─ KR3: 평균 응답 시간을 24시간 이내로 단축
```

---

## 📝 "Add New Objective" 입력 필드 설명

### 1. **Objective Title** (필수 ⭐)

**의미:**
- 달성하고자 하는 **목표의 제목**
- 팀이나 개인이 해당 분기에 집중할 핵심 목표

**목적:**
- 명확하고 이해하기 쉬운 목표 설정
- 팀원 간 목표 공유 및 정렬
- 조직 전체의 방향성 통일

**예시:**
- ✅ "제품의 시장 적합성 증대"
- ✅ "브랜드 인지도 2배 향상"
- ✅ "영업팀 생산성 극대화"
- ❌ "열심히 일하기" (모호함)
- ❌ "매출 100만원 달성" (정량적 → Key Result로 이동)

**작성 팁:**
- 동사형으로 시작 ("증대", "향상", "개선")
- 측정 가능한 숫자는 Key Result에 작성
- 간결하고 영감을 주는 문장 (1줄)

---

### 2. **Description** (선택)

**의미:**
- 목표에 대한 **상세한 설명 및 배경**
- 왜 이 목표가 중요한지, 어떤 맥락에서 설정되었는지

**목적:**
- 목표의 배경과 중요성 공유
- 새로운 팀원이 맥락을 빠르게 이해
- 분기 종료 후 회고 시 참고 자료

**예시:**
```
현재 고객 이탈률이 20%로 높은 수준이며, 
경쟁사 대비 제품 만족도가 낮은 상황입니다.
고객 경험을 개선하여 장기 고객 확보를 목표로 합니다.
```

**작성 팁:**
- 현재 상황 (As-is)
- 목표 상황 (To-be)
- 중요한 이유 (Why)
- 2~3문장으로 간결하게

---

### 3. **Quarter** (필수 ⭐)

**의미:**
- 목표를 달성할 **분기** (3개월 단위)
- 2024-Q1, 2024-Q2, 2024-Q3, 2024-Q4 등

**목적:**
- 단기 목표 설정으로 집중력 향상
- 분기별 성과 측정 및 비교
- 연간 로드맵 수립

**OKR에서 분기가 중요한 이유:**
1. **짧은 주기:** 3개월은 변화를 만들기에 적절한 기간
2. **빠른 피드백:** 분기마다 목표를 재조정 가능
3. **민첩성:** 시장 변화에 빠르게 대응

**선택 방법:**
- 현재 분기 또는 다음 분기 선택
- 장기 목표는 여러 분기로 분할

---

### 4. **Team** (선택)

**의미:**
- 목표를 담당하는 **팀 이름**
- 예: Product, Marketing, Sales, Engineering

**목적:**
- 팀별 목표 관리
- 팀 간 협업 목표 식별
- 조직 전체 성과 분석 시 팀 단위 집계

**예시:**
- "Product Team"
- "Marketing"
- "Sales"
- "Customer Success"
- "Engineering - Backend"

**활용 방법:**
- 팀 목표: 팀 전체가 협력하는 목표
- 개인 목표: 팀 이름 + 개인 이니셜 (예: "Product - JD")

---

### 5. **Owner** (선택)

**의미:**
- 목표에 대한 **최종 책임자** (담당자)
- 한 명의 명확한 주인(Owner)

**목적:**
- 책임 소재 명확화 (Accountability)
- 진행 상황 추적 시 연락처
- 의사결정 권한자 식별

**예시:**
- "John Doe" (개인 이름)
- "김철수"
- "Product Manager"

**OKR 원칙:**
- **단일 책임자:** 목표당 1명의 Owner 권장
- **권한 부여:** Owner에게 의사결정 권한 제공
- **협업자 구분:** Owner ≠ 실무 담당자 (Key Result에 별도 Owner)

---

### 6. **Status** (필수 ⭐)

**의미:**
- 목표의 **현재 진행 상태**
- 실시간으로 목표 달성 가능성 평가

**목적:**
- 위험 신호 조기 감지
- 경영진 보고용 대시보드
- 팀 간 현황 공유

#### 📊 Status 항목 상세 설명

| Status | 한글 | 색상 | 의미 | 사용 시점 |
|--------|------|------|------|-----------|
| **On Track** | 정상 진행 중 | 🟢 녹색 | 목표가 계획대로 진행되고 있으며, 분기 종료 시 달성 가능 | 진척률 75% 이상 |
| **At Risk** | 위험 | 🟠 주황색 | 약간의 지연이 있거나 위험 요소가 존재하지만, 조치를 취하면 달성 가능 | 진척률 50~75% |
| **Behind** | 지연됨 | 🔴 빨간색 | 목표 달성이 어려운 상황이며, 즉각적인 조치 또는 목표 재조정 필요 | 진척률 50% 미만 |
| **Completed** | 완료됨 | 🔵 파란색 | 모든 Key Results가 100% 달성되어 목표 완료 | 진척률 100% |

#### 🎯 Status 선택 가이드

**🟢 On Track (정상 진행 중)**
```
✅ 모든 Key Results가 예상 진척률 달성
✅ 특별한 장애물 없음
✅ 팀의 사기가 높음
✅ 추가 리소스 불필요

예시: "KR1: 50/50 달성 (100%), KR2: 45/50 달성 (90%)"
```

**🟠 At Risk (위험)**
```
⚠️ 일부 Key Results가 예상보다 느림
⚠️ 외부 요인(예산, 인력)으로 인한 불확실성
⚠️ 해결 가능한 장애물 존재
⚠️ 관리자의 주의가 필요한 상태

예시: "KR1: 30/50 달성 (60%), 예산 승인 대기 중"
```

**🔴 Behind (지연됨)**
```
❌ 대부분의 Key Results가 크게 미달
❌ 구조적 문제 또는 예상치 못한 위기
❌ 팀 내 심각한 이슈 (이직, 충돌)
❌ 목표 재설정 또는 리소스 재배치 필요

예시: "KR1: 10/50 달성 (20%), 주요 개발자 이직으로 프로젝트 중단"
```

**🔵 Completed (완료됨)**
```
✅ 모든 Key Results 100% 달성
✅ 분기 종료 전에도 조기 완료 가능
✅ 성과 인정 및 팀 축하 필요
✅ 다음 분기 목표 설정 준비

예시: "KR1: 50/50 (100%), KR2: 55/50 (110%) 달성!"
```

#### 💡 Status 업데이트 주기

| 주기 | 권장 사항 |
|------|-----------|
| **주간 (Weekly)** | 팀 내부 체크인 시 Key Result 진척률 기반으로 업데이트 |
| **격주 (Bi-weekly)** | 관리자와 1:1 미팅 시 Status 리뷰 |
| **월간 (Monthly)** | 경영진 보고 시 전체 OKR Status 리포트 |

---

### 7. **Start Date** (선택)

**의미:**
- 목표 시작 날짜

**목적:**
- 타임라인 관리
- 분기 시작일과 다를 경우 명시
- 프로젝트 일정 추적

**기본값:**
- 보통 분기 시작일 (1월 1일, 4월 1일, 7월 1일, 10월 1일)

---

### 8. **End Date** (선택)

**의미:**
- 목표 종료 날짜 (데드라인)

**목적:**
- 명확한 기한 설정
- 긴박감 조성 (Sense of Urgency)
- 진척률 계산 기준

**기본값:**
- 보통 분기 종료일 (3월 31일, 6월 30일, 9월 30일, 12월 31일)

**주의사항:**
- End Date가 지나면 자동으로 회고 시작
- 연장이 필요한 경우 새로운 Objective 생성 권장

## 🔄 워크플로우

### 1. Objective 생성
```
Add New Objective → 정보 입력 → Create
```

### 2. Key Results 추가
```
Objective 확장 → Add Key Result → 측정 지표 입력
```

### 3. 진척률 업데이트
```
Update Progress 버튼 → Current 값 업데이트 → 자동 계산
```

### 4. Status 업데이트
```
주간/월간 체크인 시 Status 변경
```

## 🎓 OKR 작성 베스트 프랙티스

### ✅ 좋은 Objective 예시

| Objective | 설명 |
|-----------|------|
| "글로벌 시장 점유율 확대" | 명확하고 영감을 주는 목표 |
| "제품 안정성을 업계 최고 수준으로 개선" | 비교 기준 명시 |
| "고객 경험을 혁신하여 NPS 리더가 된다" | 정성적 목표 + KR에서 정량화 |

### ❌ 나쁜 Objective 예시

| Objective | 문제점 |
|-----------|--------|
| "매출 1억 달성" | 너무 구체적 (KR로 이동) |
| "열심히 일하기" | 측정 불가능, 모호함 |
| "버그 수정 100개" | 단순 작업 나열 (목표가 아님) |

### 💡 Key Results 작성 팁

**SMART 원칙:**
- **S**pecific (구체적): "고객 만족도 향상" ❌ → "NPS 50점 달성" ✅
- **M**easurable (측정 가능): 숫자로 표현
- **A**chievable (달성 가능): 도전적이지만 현실적
- **R**elevant (관련성): Objective와 직접 연관
- **T**ime-bound (기한 명시): 분기 내 달성

       Current: 8,500 / Target: 10,000 / Unit: users

## 🚨 자주 묻는 질문 (FAQ)

### Q1: Objective와 Key Result의 차이는?
**A:** 
- **Objective:** "무엇을" 달성할지 (정성적 목표)
- **Key Result:** "어떻게" 측정할지 (정량적 지표)

### Q2: 한 분기에 몇 개의 Objective를 설정해야 하나요?
**A:** 
- **개인:** 2~3개 권장
- **팀:** 3~5개 권장
- **회사:** 3~5개 권장
- 너무 많으면 집중력 분산!

### Q3: Status는 누가 업데이트하나요?
**A:** 
- **Owner** (담당자)가 주기적으로 업데이트
- 주간 팀 미팅 시 함께 리뷰 권장

### Q4: Key Result를 100% 달성 못하면 실패인가요?
**A:** 
- OKR은 **도전적 목표** 설정이 목적
- 70~80% 달성도 성공으로 간주
- 매번 100% 달성하면 목표가 너무 낮다는 신호

### Q5: 분기 중간에 Objective를 변경할 수 있나요?
**A:** 
- 가능하지만 **권장하지 않음**
- 단, 외부 환경이 크게 변한 경우 (시장 변화, 위기 등) 조정 가능
- 변경 시 팀과 충분한 논의 필요
