# 서비스 기획서 — Proce (Decision OS)

## 1. 프로젝트 개요

### 1.1 서비스명

**Proce**

### 1.2 서비스 한줄 소개

**"회의·보고·결재를 데이터·정책 기반으로 대체하는 AI 의사결정 운영체계"**

### 1.3 서비스 목적 및 목표

조직의 모든 산출물을 단일 데이터 허브로 모아 AI와 정책(DSL) 기반으로 공정하고 빠른 의사결정을 실행합니다.

- **핵심 목표:**
  - **관리 병목 축소**: 사람 중심 결재 → 정책 기반 자동 승인/보류/거절
  - **의사결정 자동화**: 입력 → 분석 → 제안 → 승인 → 작업 생성의 완전 자동화
  - **데이터 일원화**: 산출물/피드백/KPI/로그를 통합 구조로 집계
  - **공정성 제고**: 익명 Alias로 편향 최소화
  - **역할 확장**: 부서 경계를 넘는 기여 가능
  - **자가 학습 조직화**: 데이터 누적에 따른 지속적 개선

### 1.4 타겟 사용자 분석

| 구분 | 주요 사용자 | 핵심 목적 및 흐름 |
| --- | --- | --- |
| **의사결정자** | 임원/팀리드 | 정책 기준으로 자동 승인·보류, 에스컬레이션 모니터링 |
| **실무자** | 전 직원 | 자유 입력 → AI 정규화 → KPI/작업 연결 → 결과 피드백 |
| **운영/기획** | PM/OPS | 정책(DSL) 설계, KPI 체계 관리, 효과 측정 |

## 2. 시장 분석

### 2.1 시장 현황
- 비동기 협업/생산성 도구 확산, 데이터 기반 의사결정 전환 가속
- 회의/보고/결재의 비효율 문제 장기화 → 자동화 수요 증가

### 2.2 경쟁사 분석
- **협업툴 계열**: Slack/Notion/Jira — 기록·업무관리 중심, 의사결정 자동화는 제한적
- **BI/분석 계열**: Tableau/Looker — 인사이트 제공, 정책 실행과 워크플로우 자동화는 약함
- **LLM 활용툴**: 다양한 AI 요약/자동화 툴 — 단편 자동화, 조직 운영체계 수준의 일관성 부족

### 2.3 차별화 포인트
1. **NoMeet Engine**: 회의 대체형 비동기 구조화 토론 플로우
2. **Policy Decision DSL**: 규칙/근거/신뢰도/추적성까지 내장
3. **Reporting-Free Input**: 형식 자유 입력 → AI가 정규화 및 연결
4. **공정성/익명성**: Alias 기반 평등한 평가 체계

## 3. 핵심 기능 시스템

### 3.1 주요 기능 정의

#### 기능 1: NoMeet Engine
- 목적: 동기 회의 의존도 제거, 의사결정 속도 향상
- 주요 특징: Agenda → AI Briefing → Feedback → AI Proposal → Execution
- 기대 효과: 회의 시간 40~60% 절감

#### 기능 2: Policy Decision Engine (DSL)
- 목적: 정책 기반 자동 승인/보류/거절
- 주요 특징: 증거 링크·신뢰도·추적성 로깅, 에스컬레이션
- 기대 효과: 결재 리드타임 2~3배 단축

#### 기능 3: Reporting-Free Input
- 목적: 보고 문서 작성 시간 제거, 산출물 직접 연결
- 주요 특징: 자유 입력 → AI 추출/정규화 → KPI/Task 연결
- 기대 효과: 보고 시간 70~80% 절감

#### 기능 4: Fair & Anonymous Collaboration
- 목적: 편향 최소화, 공정 평가
- 주요 특징: Alias 표시, 콘텐츠 가치 중심 평가

#### 기능 5: Centralized Performance OS
- 목적: 조직 지식 그래프 축적, 자동 성과 분석
- 주요 특징: 산출물/피드백/결정/근거 데이터의 통합

### 3.2 기술 요구사항

#### 프론트엔드
- 기술 스택: React 19 + Vite + TypeScript + Tailwind v4
- 상태 관리: TanStack Query + Context API
- 스타일링: Promptor 디자인 시스템 적용 (Primary #3D3EFF)

#### 백엔드 (후속 단계)
- API/권한/RBAC, 데이터 레이크, RAG 엔진, 정책 DSL 실행기 (별도 정의)

## 4. 사용자 여정 (User Journey)

### 4.1 사용자 페르소나
- 의사결정자, 실무자, 운영/기획 3그룹

### 4.2 사용자 플로우 (요약)
1. 자유 입력 → AI 정규화 → KPI/Task 연결
2. NoMeet 스레드 참여 → AI 요약/제안 확인 → 승인/실행
3. 정책 엔진 자동 승인/에스컬레이션 → 실행 로그/지표 반영

### 4.3 사용자 시나리오
- 시나리오 A: 신규 이슈 제기 → AI 요약 → 합의 → 작업 생성
- 시나리오 B: KPI 편차 감지 → 정책 평가 → 교정 액션 자동 배치
